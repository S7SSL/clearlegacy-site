## Known Quirks

- **Wrangler from `worker/` directory tries to scan parent for assets**, fails with "Asset too large" on `worker/node_modules/workerd/bin/workerd` (104 MiB). Workaround: deploy from `/tmp/cl-worker-deploy` (clean copy of `worker/` outside `clearlegacy-site/`). Re-create with `cp -r ~/clearlegacy-site/worker /tmp/cl-worker-deploy && cd /tmp/cl-worker-deploy && rm -rf node_modules .wrangler _empty_assets/placeholder.txt && npm install` if `/tmp` was cleared.
- **Local `git checkout main; git merge add-funnel-events`** sequence can fail with "uncommitted local changes" because of working-tree state in `worker/`. Workaround used today: open a Pull Request on GitHub UI and merge there.
- **`.uetbak` files** — every HTML modified by today's UET install has a `.uetbak` sibling backup (gitignored). To roll back UET, find/rename them.
- **Two pre-existing TS errors** in `worker/src/handlers/webhook.ts` were fixed today (`generatingStartedAt`, `appendActivity`).
