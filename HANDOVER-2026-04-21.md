## Root cause found

PDF generation fails at the **email** step. From the lead record:

```
pdfError: "Resend send failed: 401 {\"statusCode\":401,\"name\":\"validation_error\",\"message\":\"API key is invalid\"}"
```

The **RESEND_API_KEY** secret currently deployed to the Worker is rejected by Resend. Everything upstream of the email step works:

| Pipeline step | Status |
|---|---|
| /api/lead (validate + store + return Stripe URL) | working |
| Browser Rendering (HTML → PDF) | working — produced a 77 kB PDF |
| R2.put (`wills/9e03f03e-...pdf`) | working — object is in the bucket |
| Resend email send | **FAILING (401 invalid key)** |
| updateLead pdfStatus:ready + emailedAt | not reached |

Because the email step throws, the catch block flips `pdfStatus` to `failed` without ever writing `pdfKey`/`pdfGeneratedAt` to the lead record — but the PDF itself is safely in R2.
